# 写作路线图

这是一份边学边写的 LLM 工程学习笔记。路线图会随着学习推进持续调整，优先保证每个主题讲清楚、能运行、可复现。

## 写作原则

- 先建立直觉，再进入公式和代码。
- 每个核心概念尽量配一个最小可运行实验。
- 记录学习过程中的误解、卡点和工程细节。
- 明确区分已经完成、正在写和计划中的内容。

## 每篇文章的结构

1. 核心一句话：这个技术解决什么问题。
2. 背景与动机：为什么需要它。
3. 直觉解释：先不用公式建立图景。
4. 数学定义：把关键公式讲清楚。
5. 最小代码：用可运行代码验证直觉。
6. 工程注意事项：记录真实项目里的坑。
7. 延伸阅读：论文、博客、视频和源码。
