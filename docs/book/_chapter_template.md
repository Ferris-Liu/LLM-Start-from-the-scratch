# 第 X 章：标题

## 1. 本章要解决的问题

- 本章试图回答什么问题？
- 为什么这个问题重要？
- 它和前后章节是什么关系？

## 2. 你学完后应该会什么

- 能用自己的话解释核心概念
- 能看懂关键公式或流程图
- 能完成一个最小实验或代码实现
- 能回答对应面试问题

## 3. 核心直觉

先用通俗语言解释核心概念。

可以包括：

- 一个生活类比
- 一个最小例子
- 一个“如果没有它会怎样”的反例

## 4. 数学定义

列出必要公式，并解释：

- 每个符号是什么意思
- 输入输出分别是什么
- 为什么这个公式合理

如果数学细节太多，可以只保留最核心的一层。

## 5. 从直觉到实现

这一节用流程而不是公式堆砌，解释实际系统里它是怎么工作的。

例如：

1. 输入是什么
2. 中间发生了什么
3. 输出是什么
4. 训练时怎么用
5. 推理时怎么用

## 6. 最小代码实现

给出最小可运行代码，尽量短。

建议结构：

- 环境说明
- 核心代码
- 输入输出示例
- 运行结果解释

## 7. 实验观察

记录：

- 输入
- 输出
- loss / metric
- 现象分析

重点写：

- 哪些结果符合预期
- 哪些结果不符合预期
- 可能原因是什么

- 误区 1
- 误区 2
- 误区 3

每条误区后面最好补一句“为什么这其实不对”。
